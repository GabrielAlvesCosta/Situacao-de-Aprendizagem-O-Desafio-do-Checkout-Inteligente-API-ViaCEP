CT-01 — Consulta de CEP Válido e Existente

Campo Detalhe
ID CT-01
Função testada test_cep_valido_retorna_200_e_dados
Objetivo Validar se a API retorna os dados corretamente para preencher o formulário HTML
Entradas cep = "01001000"
Resultado esperado Status 200 e dados do endereço (Praça da Sé)
Técnica Igualdade direta e ausência de chave (assert ==, assert not in)
Tipo Caminho Feliz 
def test_cep_valido_retorna_200_e_dados(api_client, base_url):
    cep = "01001000"
    url = f"{base_url}{cep}/json/"
    
    response = api_client.get(url)
    dados = response.json()
    
    assert response.status_code == 200
    assert "erro" not in dados
    assert dados["cep"] == "01001-000"
    assert dados["logradouro"] == "Praça da Sé"
    assert dados["bairro"] == "Sé"
    assert dados["localidade"] == "São Paulo"

CT-05 — Verificação de Contrato (Schema JSON)

Campo Detalhe
ID CT-05
Função testada test_estrutura_do_json_retornado_schema
Objetivo Garantir que a API externa não alterou a estrutura das chaves do objeto
Entradas cep = "01001000"
Resultado esperado Todas as chaves obrigatórias presentes no dicionário retornado
Técnica Laço de repetição e contenção (assert in)
Tipo Validação Estrutural / Schema 
def test_estrutura_do_json_retornado_schema(api_client, base_url):
    cep = "01001000"
    url = f"{base_url}{cep}/json/"
    
    response = api_client.get(url)
    dados = response.json()
    
    chaves_esperadas = [
        "cep", "logradouro", "complemento", "bairro", 
        "localidade", "uf", "ibge", "gia", "ddd", "siafi"
    ]
    
    assert response.status_code == 200
    for chave in chaves_esperadas:
        assert chave in dados, f"A chave obrigatória '{chave}' sumiu do retorno!"

CT-06 — Desempenho e SLA de Resposta

Campo Detalhe
ID CT-06
Função testada test_tempo_de_resposta_da_api
Objetivo Validar se o tempo de resposta garante uma boa experiência ao usuário no front-end
Entradas cep = "01001000"
Resultado esperado Tempo de requisição menor que 2 segundos
Técnica Comparador de limite (assert <)
Tipo Teste de Performance / NFR (Requisito Não Funcional) 
def test_tempo_de_resposta_da_api(api_client, base_url):
    cep = "01001000"
    url = f"{base_url}{cep}/json/"
    
    response = api_client.get(url)
    tempo_resposta = response.elapsed.total_seconds()
    
    assert tempo_resposta < 2.0, f"A API demorou muito! Tempo: {tempo_resposta}s"

Resumo das técnicas utilizadas

Técnica Quando usar
assert valor1 == valor2 Comparação de valores exatos (status HTTP, retornos de texto e booleanos)
assert chave in objeto Garantir que uma propriedade existe dentro da estrutura do JSON de resposta
assert chave not in objeto Garantir que o JSON retornado em sucesso está livre de flags de falha
assert tempo < limite, "msg" Verificações de limite com mensagens customizadas para debug fácil em testes de performance

Como executar os testes

A partir da raiz do projeto, instale os requisitos caso seja a primeira vez e execute o pytest:

# Rodar todos os testes com saída simplificada

# Rodar detalhadamente para visualizar o resultado de cada caso
pytest -v

# Rodar um teste de falha/performance específico para debug
pytest tests/test_viacep.py::test_tempo_de_resposta_da_api
